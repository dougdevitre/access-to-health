# Partner Onboarding Guide

> For organizations wanting to adopt, adapt, or contribute to Access to Health
> Hospitals, FQHCs, community organizations, academic institutions, and government agencies

---

## Who Can Use This

| Partner Type | What You Get | What You Contribute |
|-------------|-------------|---------------------|
| **Hospitals / Health Systems** | CHNA alignment, community benefit templates, SDOH screening protocols | Local data, clinical workflows, community benefit funding |
| **FQHCs** | CHW workflows, screening tools, referral protocols, reporting templates | Frontline implementation experience, patient-level outcomes |
| **Community Organizations** | Population deep dives, engagement strategies, advocacy toolkit | Community voice, cultural expertise, trust networks |
| **Academic Institutions** | Evaluation frameworks, research questions, training curricula | Research capacity, student workforce, publication |
| **Local Government** | HIA toolkit, policy templates, data sources, constituent comms | Political support, cross-sector coordination, funding |
| **State Health Departments** | Scalable framework, training materials, data standards | Statewide data, technical assistance, funding pass-through |
| **Technology Companies** | Requirements specs, data schemas, compliance guides | Platform development, integration, technical support |

## Getting Started (4 Steps)

### Step 1: Assess Fit (Week 1)

Read these files to understand the system:
1. `SKILL.md` — Core routing loop
2. `NAVIGATION.md` — Find content for your role
3. `roles/ROLE-REGISTRY.md` — Map your staff to the 20 roles
4. `populations/POPULATION-REGISTRY.md` — Identify your priority populations

**Key question**: Which parts of Access to Health address your current gaps?

### Step 2: Adapt (Weeks 2-4)

| Task | How |
|------|-----|
| **Localize resources** | Update `features/resource-navigator.md` with your local resources |
| **Adapt for your state** | Use `references/state-adaptation-template.md` to create your state reference |
| **Customize templates** | Modify `artifacts/reporting-templates.md` for your reporting requirements |
| **Configure workflows** | Adjust timelines and escalation in `workflows/` to match your capacity |
| **Brand communications** | Customize `messaging/` and `communication/` templates with your org name |

### Step 3: Train (Weeks 3-6)

| Audience | Training Content | Duration |
|----------|-----------------|----------|
| Leadership | `roles/hdo.md` + `integration/fiscal-operations.md` | 2 hours |
| CHWs | `roles/chw.md` + `features/sdoh-screener.md` | 4 hours |
| Clinical staff | `roles/phn.md` + `features/resource-navigator.md` | 3 hours |
| Communications | `communication/` playbooks + `messaging/` | 3 hours |
| Data/IT | `integration/developer-guide.md` + `schemas/` | 4 hours |
| All staff | `features/education-toolkit.md` Module 4 (PH 101) | 2 hours |

### Step 4: Implement and Contribute Back

- Start with 1-2 workflows and expand
- Track what works and what doesn't
- Submit improvements back:
  - New local resources → update `features/resource-navigator.md`
  - State-specific data → create state reference file
  - Cultural insights → expand population deep dives
  - Better templates → improve `artifacts/`
  - Bug fixes → use GitHub issue templates

## MOU Template for Partnerships

```
MEMORANDUM OF UNDERSTANDING
Access to Health Implementation Partnership

Between: [Your Organization] and [Partner Organization]

PURPOSE
Implement the Access to Health framework to improve [specific health outcomes]
for [target population] in [geography].

SCOPE
- Adopt [specific components] from the Access to Health system
- Adapt for local context (resources, data, regulations)
- Train [#] staff across [roles]
- Implement for [duration]
- Evaluate outcomes and contribute improvements

ROLES
[Your Organization] will:
1. Provide implementation leadership and project management
2. Adapt content for local context
3. Train staff
4. Collect and report outcome data

[Partner Organization] will:
1. [Specific contribution — data, funding, clinical access, community reach]
2. [Specific contribution]
3. Participate in evaluation

DATA SHARING
- Aggregate, de-identified data shared for system improvement
- Individual-level data handled per HIPAA / 42 CFR Part 2
- Data sharing agreement attached as Appendix A

DURATION: [Start] through [End]
REVIEW: Quarterly joint review meetings

ATTRIBUTION
Both parties will attribute the Access to Health framework (MIT License)
in publications, presentations, and reports.
```

## Contribution Priorities

We especially welcome contributions in these areas:

### High Priority
- [ ] State adaptations beyond Missouri (legal authority, data sources, organizations)
- [ ] Additional language layers (Arabic, Somali, Mandarin, Vietnamese, French)
- [ ] Real-world implementation case studies
- [ ] Updated statistics and data (please include source and date)
- [ ] Accessibility improvements for the documentation site

### Medium Priority
- [ ] Additional population deep dives (currently 25 profiles, 25 individual files)
- [ ] New reporting templates for specific grant programs
- [ ] Integration guides for specific EHR systems (Epic, Cerner, Athena)
- [ ] Additional developer tools and tests
- [ ] Video/multimedia versions of training content
